## Description

The chat view does not scroll when messages overflow the viewport. The root cause is that the chat page uses a React fragment (`<>`) as its root element, which breaks the flex column layout needed for the ScrollArea in MessageList to receive a constrained height. Without a flex container parent, `flex-1` on MessageList has no effect and the content grows unbounded instead of scrolling.

## Acceptance Criteria

- [x] Chat message list scrolls when messages exceed viewport height
- [x] Chat input remains fixed at the bottom
- [x] Auto-scroll to bottom on new messages still works

## Implementation Steps

- [x] Replace React fragment with `<div className="flex h-full flex-col">` in chat page
- [x] Add `overflow-hidden` to ScrollArea root to constrain content
